# Tech Benefit Radar

동동이 놓치고 싶지 않은 모집/혜택/굿즈/크레딧 이벤트를 추적하기 위한 레이더 문서.

## 목표
- AI/개발/클라우드 관련 모집, 프로그램, 해커톤, 학생 혜택을 조기에 발견
- 굿즈, 수료 혜택, 크레딧, 무료 플랜, 베타 우선권이 있는 공고를 우선 추적
- 동동이 이미 참여한 이벤트는 중복 알림하지 않음
- 새로운 기획/새 시즌/신규 모집이 나왔을 때만 알림

## 알림 원칙
1. **새 기획 / 새 시즌 / 신규 모집을 우선 알림**
2. **이미 참여한 동일 프로그램은 기본적으로 반복 알림하지 않음**
3. 단, 아래 경우는 재알림 가능
   - 마감 임박
   - 혜택이 큼 (굿즈/크레딧/무료 플랜/수료 보상)
   - 동동이 놓쳤을 가능성이 큼
   - 새 시즌/새 조건/새 혜택으로 실질적 변화가 있음
4. 아래 요소가 있으면 우선순위 상승
   - 굿즈
   - 클라우드/API 크레딧
   - 무료 플랜 / 학생 혜택
   - 수료 배지 / 참가 키트
   - 얼리 액세스 / 베타 참여권
5. 한국 대상, 온라인 참여 가능, 문턱 낮은 프로그램 우선

## 체크 포인트
새 공고를 발견하면 아래를 확인해서 알려준다.
- 무엇인지
- 주최/브랜드
- 신청 마감일
- 참여 대상
- 굿즈/크레딧/사은품 여부
- 동동이 이미 참여한 동일 프로그램인지 여부
- 지금 바로 신청해야 하는지

## 재알림 규칙
- 기본적으로 같은 프로그램은 반복 알리지 않음
- 단, 아래 조건이면 다시 알림
  - 마감 임박 (예: 3일 이내)
  - 새 시즌 / 새 모집 / 새 혜택 추가
  - 굿즈 / 크레딧 / 무료 플랜 / 참가 보상 가치가 큼
  - 동동이 놓쳤을 가능성이 큼

## 우선 알릴 이벤트 타입
- 굿즈 제공
- 클라우드/API 크레딧 제공
- 학생/개발자 무료 플랜
- 수료 보상 / 참가 키트
- 해커톤 / 챌린지 / 스터디 모집
- 한국 대상 또는 온라인 참여 가능 이벤트

## 운영 판단 기준
새 이벤트를 발견했을 때 아래 셋 중 하나로 분류한다.
- `ALERT_NOW`: 지금 바로 알려야 함
- `REMIND_LATER`: 중요하지만 아직 급하지 않음
- `IGNORE`: 가치 낮거나 이미 충분히 알려진 것

`ALERT_NOW` 조건 예시:
- 마감이 임박했고 혜택이 큼
- 동동 취향(굿즈/크레딧/무료 플랜)에 정확히 맞음
- 새 시즌/새 혜택으로 이전과 다른 실질 가치가 생김

## 추적 우선순위
### 최우선
- 신규 오픈
- 마감 임박
- 굿즈/크레딧 제공
- 학생/개발자 무료 혜택

### 중간
- 일반 교육 프로그램
- 커뮤니티성 이벤트

### 낮음
- 혜택 없는 단순 홍보성 공지
- 이미 참여한 프로그램의 반복 홍보

## 알림 형식
동동에게 보낼 때는 짧게:
- 프로그램명
- 왜 중요한지
- 마감 언제인지
- 굿즈/크레딧/혜택 여부
- 이미 참여한 것과 다른 점

예시:
- "새 Google Cloud Study Jam 열림. 이번 시즌은 Gemini 실습 + 수료 굿즈 있음. 마감 4/14. 지난번이랑 다른 시즌이라 신청 가치 있음."

## 운영 메모
- 앞으로 비슷한 이벤트를 발견하면 이 문서를 기준으로 중복 여부 판단
- 필요하면 heartbeat/정기 점검 태스크로 확장
